# Deploying Master Numerics (Render)

This repo ships a **Render Blueprint** (`render.yaml`) that deploys the whole
app as **one web service** — Express serves the API + Socket.io battles **and**
the built React client — alongside managed **PostgreSQL** and **Redis**.

It launches in **free-demo mode**: the paywall is off (everyone can sign up and
use everything) and Stripe/email are optional. You can turn on payments later.

---

## 1. One-time deploy

1. Push this branch to GitHub (already done) — or merge it into your default
   branch. If you deploy a branch other than `claude/zealous-mayer-6qrsqn`,
   edit the `branch:` line in `render.yaml` first.
2. Go to **https://dashboard.render.com → New → Blueprint**.
3. Connect this GitHub repo. Render detects `render.yaml` and shows three
   resources: the **web service**, **Postgres**, and **Redis**.
4. Click **Apply**. Render provisions the database + Redis, generates the JWT
   secrets, wires all env vars, builds the client + server, runs the DB
   migration + seed, and starts the app.
5. When the web service is **Live**, open its URL
   (e.g. `https://master-numerics.onrender.com`). Register an account (1-day
   trial, but the paywall is off so access never lapses) and you're in.

First request after idle may **cold-start** (~30–60s) on the free plan.

### Free-plan caveats
- Free web services **spin down** after inactivity (cold starts).
- A free Postgres instance is deleted after ~30 days — upgrade to a paid
  instance ($7/mo) for anything you want to keep.
- The build runs `prisma db push` + an **idempotent seed** on boot; the seed
  **skips** if content already exists, so user data is never wiped on redeploy.
  (Set `FORCE_RESEED=1` to rebuild content.)

## 2. Make yourself an admin

The admin panel (`/admin`) is role-gated. After registering:

1. Render → your **Postgres** → **Connect** → open a `psql` shell (or use the
   external connection string locally), then:
   ```sql
   UPDATE "User" SET role = 'ADMIN' WHERE email = 'you@example.com';
   ```
2. **Sign out and back in** (the role is baked into the access token).
   An **Admin** link now appears in the dashboard header.

---

## 3. Turn on payments later (optional)

To enable the $4.99/mo · $39.99/yr Stripe paywall:

1. In Stripe, create the two recurring **Prices** and copy their IDs.
2. Add a webhook endpoint → `https://<your-url>/api/billing/webhook`, and copy
   its **signing secret**.
3. On the Render web service → **Environment**, set:
   - `PAYWALL_ENABLED=true`
   - `STRIPE_SECRET_KEY=sk_live_…` (or `sk_test_…`)
   - `STRIPE_WEBHOOK_SECRET=whsec_…`
   - `STRIPE_PRICE_MONTHLY=price_…`
   - `STRIPE_PRICE_YEARLY=price_…`
   - `CLIENT_ORIGIN=https://<your-url>` (used for checkout redirects)
4. Save → Render redeploys. The trial-then-paywall flow is now live.

**Email** (password resets) is optional too — set `SMTP_HOST`, `SMTP_PORT`,
`SMTP_USER`, `SMTP_PASS`, `EMAIL_FROM` to enable real emails (otherwise reset
links are logged to the server console).

---

## 4. Custom domain

Render web service → **Settings → Custom Domains** → add your domain and follow
the DNS instructions. Then update `CLIENT_ORIGIN` to the new URL.

---

## 5. Run locally

```bash
docker compose up -d            # Postgres + Redis
cd server && npm install && cp .env.example .env   # fill DATABASE_URL etc.
npx prisma db push && npm run db:seed && npm run dev
cd ../client && npm install && npm run dev          # http://localhost:5173
```

The client dev server proxies `/api` and `/socket.io` to the backend on `:4000`.

## Environment variables (reference)

| Var | Required | Notes |
|---|---|---|
| `DATABASE_URL` | yes | wired by the blueprint |
| `REDIS_URL` | yes | wired by the blueprint |
| `JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` | yes | auto-generated by the blueprint |
| `PAYWALL_ENABLED` | no (default `false`) | `true` to require trial/subscription |
| `CLIENT_DIST` | no | path to built client (set by blueprint) |
| `CLIENT_ORIGIN` | no | defaults to the Render URL; set for Stripe redirects |
| `STRIPE_*` | only if paywall on | secret key, webhook secret, two price IDs |
| `SMTP_*`, `EMAIL_FROM` | no | enables real password-reset emails |
